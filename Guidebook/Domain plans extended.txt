DOMAIN 9 — CONFIDENCE & SOURCE METADATA

Purpose:
Track why something is believed, how strongly, and under what conditions it may change.

This domain never alters facts. It annotates them.

Design principles

• Multiple sources per claim
• Confidence is scoped (climate, region, context)
• Distinguish empirical evidence vs horticultural consensus
• Support future audits and corrections

DOMAIN 10 — AI REASONING RULES

Purpose:
Turn static knowledge into explainable decisions, not black-box predictions.

No guessing.
No probabilistic hallucinations.
Rules are explicit, inspectable, and overridable.

Rule taxonomy

• Eligibility rules (can this plant grow here?)
• Recommendation rules (should it be planted now?)
• Diagnostic rules (what’s likely wrong?)
• Intervention rules (what action helps now?)

JSON
{
  "reasoning_rules": [
    {
      "id": "rule_sowing_season_match",
      "type": "recommendation",
      "description": "Recommend sowing only if current month is within sowing window",
      "inputs": [
        "current_month",
        "plant_id",
        "climate_zone_id"
      ],
      "conditions": [
        {
          "fact": "sowing_calendar.sowing_window",
          "operator": "contains",
          "value": "current_month"
        }
      ],
      "outputs": {
        "recommendation": "allow_sow",
        "confidence_modifier": 1.0
      },
      "explanation_template": "This plant is typically sown during this month in your climate zone."
    },
    {
      "id": "rule_frost_sensitive_block",
      "type": "constraint",
      "description": "Prevent sowing frost-sensitive plants when frost risk exists",
      "conditions": [
        {
          "fact": "plant_climate_requirements.frost_tolerant",
          "operator": "equals",
          "value": false
        },
        {
          "fact": "environment.frost_risk",
          "operator": "equals",
          "value": true
        }
      ],
      "outputs": {
        "recommendation": "block_sow"
      },
      "explanation_template": "This plant is sensitive to frost, which is still likely in your area."
    }
  ]
}


Every recommendation can be traced:
rule → facts → explanation → sources.

This is critical for user trust and regulatory safety.

DOMAIN 11 — USER BEHAVIOR & FEEDBACK LOOPS

Purpose:
Let the system learn from users without redefining reality.

User data influences confidence, personalization, and heuristics, never base facts.

Design boundaries

• User feedback never overwrites authoritative data
• Feedback is contextual (location, soil, setup)
• Learning adjusts weights, not truths
