To implement the Garden Deck: Dresden Pro using an AI-agent architecture, we must define each component as a discrete "Service" or "Agent Module." This ensures the system can handle the complex logic of biological simulation while maintaining a fluid, gamified UI.

1. The Seed Vault (The "Deep Storage" Module)

The Vault is the master repository of all genetic assets owned by the user.

Logic: It acts as the "Cold Storage" for seed data. It tracks quantities, purchase dates, and expiration (viability) based on species-specific shelf-life.

Agent Role: A Librarian Agent monitors the vault, notifying the user when seeds are getting old or when "Special Varieties" (e.g., Heirloom Harzfeuer Tomatoes) are available for the current season.

UI: A heavy, secure-looking digital binder with sorting filters for "Rarity," "Origin," and "Germination Rate."

2. Seed Inventory (The "Pocket" / Active Hand)

This is the bridge between storage and the field.

Logic: When a user moves a seed from the Vault to the Inventory, it is assigned to the "Active Hand." This triggers the Pre-Planting Agent, which prepares the specific metadata needed for the current location (Dresden).

Workflow: User selects a seed packet $\rightarrow$ "Transfer to Pocket" $\rightarrow$ The seed card now floats in the Bottom Dock UI, ready for drag-and-drop.

Gamification: The "Pocket" has limited slots (e.g., 8 slots), forcing users to strategize which crops they are focusing on for the current session.

3. Sowing Calendar (The "Temporal Governance" Module)

This is the scientific heart of the app, localized for Zone 7b/8a.

Logic: A dynamic timeline mapped to the ISO 8601 week system. It calculates "Ideal," "Possible," and "Risky" windows based on historical Dresden frost dates (May 15).

Agent Role: The Chronos Agent cross-references the 200+ plant database with current calendar dates to highlight "High-Success" windows.

Visuals: Gantt-style bars showing indoor vs. outdoor sowing.

4. Real-Time Weather Integration (The "Environmental Engine")

Logic: API integration with DWD (Deutscher Wetterdienst) or OpenWeather. It pulls temperature, humidity, precipitation, and wind speed for the user's specific GPS coordinates.

Functionality: The data is fed into the Simulation Agent, which calculates "Evapotranspiration" (how much water the plants are losing) and "Frost Risk."

Workflow: If the 3:00 AM forecast shows 1¬∞C, the app sends a push notification: "Frost Alert: Cover your Basil!"

5. Gamification (The "Engagement" Layer)

This transforms a tool into a simulator.

Leveling System: Users gain "Green Thumb XP" for successful harvests, consistent watering, and identifying pests correctly.

Milestones: Unlocking "Advanced Crops" (e.g., difficult-to-grow Orchids or Melons) as the user proves their competence.

Visual Evolution: The garden grid isn't static; as plants thrive, the UI becomes more vibrant. If neglected, the UI desaturates (turns gray/brown).

6. Universal CRUD Operations (The "Omni-Editor")

Create: Add new seeds to the vault via photo/barcode scan.

Read: Deep-dive into plant history and current health stats.

Update: Edit soil types, log fertilizer applications, or change garden layouts.

Delete/Archive: Harvest a plant (move to Log) or delete a failed crop (Compost).

Agent Role: An Observer Agent records every CRUD action into a "Garden Audit Log" for end-of-season review.

7. Time Travel (The "Predictive Scrubber")

This is a high-fidelity projection feature.

Forward Travel: Scrub the timeline forward 14 days to see the "Projected Growth." The AI calculates growth based on the 7-day forecast + historical averages.

Backward Travel: Review the garden as it looked on any previous date.

UI: A "Time Dial" in the HUD. When turned, the grid "ghosts" through stages of germination and maturity.

8. Plant Care, Tips, & Hints (The "Mentor" Agent)

Logic: A contextual help system. If the user clicks a "Struggling" plant, the Mentor Agent provides a "Reasoning Payload."

Features: Pro-tips for "Thinning," "Pruning," and "Hardening Off."

Dresden Specificity: Tips like "In Dresden's sandy soil, add compost now to retain moisture."

9. Companion & Synergy Recommendations (The "Relationship" Engine)

Logic: A relational database mapping every plant to its friends and foes.

Functionality: When a seed is held over a grid slot, the system calculates a Synergy Score.

Visuals: Green/Red connection lines appear between the held seed and already-planted neighbors.

Example: Planting Marigolds next to Tomatoes reduces the "Pest Event" probability by 25%.

10. Master Plant Database (The "Knowledge Graph")

Scale: 200+ detailed profiles (Vegetables, Herbs, Flowers, Shrubs).

Search Engine: Advanced filters for "Drought Tolerant," "Bee Friendly," "Beginner Level," and "Harvest Window."

Data Points: Germination temp, pH preference, N-P-K needs, spacing (cm), and frost sensitivity.
