DOMAIN 1 — TIME & CALENDAR (FOUNDATIONAL)

This domain does not contain plant advice.
It defines how time is represented so advice can later be attached with evidence and constraints.

No guessing. No regional assumptions.

1. Calendar System (Explicit, Neutral)

We must anchor seasonality to calendar months, because “spring” is meaningless without geography.

No plant names

This is how you avoid lying to users.

DOMAIN 3 — CLIMATE & LOCATION (NORMALIZED, NON-ADVISORY)

This domain answers only one question:

“What environmental context is this garden operating in?”

Nothing more.

1. Hemispheres (Primary Temporal Modifier)

Seasons invert by hemisphere. This must be explicit.

{
  "hemispheres": [
    {
      "id": "northern",
      "description": "Northern Hemisphere"
    },
    {
      "id": "southern",
      "description": "Southern Hemisphere"
    }
  ]
}


No assumptions. Later logic will shift calendars based on this.

2. Climate Zone Systems (Declared, Not Merged)

Multiple systems exist. We do not collapse them.

{
  "climate_zone_systems": [
    {
      "id": "usda_hardiness",
      "authority": "United States Department of Agriculture",
      "basis": "Average annual minimum winter temperature",

      "id": "koppen_geiger",
      "authority": "Wladimir Köppen / Rudolf Geiger",

}


Important:
USDA ≠ growing calendar. This avoids a common lie.
